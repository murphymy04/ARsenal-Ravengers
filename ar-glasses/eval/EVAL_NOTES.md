# Speaker Classification Evaluation Notes

## What we did

Built an evaluation pipeline to measure speaker classification accuracy against hand-annotated ground truth transcripts for 3 test videos: `timur_myles_2`, `timur_peter`, `timur_will`.

### Ground truth
`eval/transcripts_to_annotate.txt` — 98 annotated segments across 3 videos with speaker labels (wearer/other). Generated by running `python -m pipeline.live` on each video, extracting combined segments, then hand-correcting speaker labels.

### Eval script
`eval_speaker_accuracy.py` — parses annotations, runs the adaptive VAD+RMS pipeline on each video, compares frame-level classifications against ground truth using per-segment majority vote. Also extracts spectral features (low/high frequency energy).

### Strategies tried

| Approach | Best error | Notes |
|---|---|---|
| Baseline adaptive (seed=0.035, alpha=0.3) | 34.7% | Current production config |
| Parameter sweep (seed/alpha/multipliers) | 30.6% | Best: seed=0.02, alpha=0.1, h=1.5, l=0.3 |
| Temporal smoothing (sliding window majority vote) | 28.6% | Window=5 frames post-classification |
| Biased boundary (boundary closer to mean_low) | 27.6% | bias=0.45 instead of 0.5 midpoint |
| Speaker inertia (require N frames to switch) | 27.6% | 8-frame inertia |
| Smoothed-RMS adaptive (average RMS before classify) | 25.5% | 5-frame RMS smoothing window |
| Segment-level adaptive (classify whole segments) | 25.5% | Mean RMS per segment vs boundary |
| Smoothed RMS + segment reclassify | 25.5% | Two-stage: frame adaptive then segment reclassify |
| Spectral features (low/high freq energy ratio) | 34.7% | Proximity effect feature, didn't help standalone |
| Combined RMS + spectral | 27.6% | Weighted normalized score, no better than RMS alone |
| Two-pass adaptive | 34.7% | Run twice, second pass seeded from first |
| Batch k-means (offline oracle) | 34.7% | k=2 on all speech RMS |
| Percentile-seeded | 33.7% | Seed from median of first 50 speech frames |

### Hard ceiling

Even with the **oracle-optimal static boundary per video** (perfect hindsight, best possible single threshold):

| Video | Optimal boundary | Segment error | Frame accuracy |
|---|---|---|---|
| timur_myles_2 | 0.008 | 3/22 = 13.6% | 79.8% |
| timur_peter | 0.044 | 7/31 = 22.6% | 78.0% |
| timur_will | 0.096 | 14/45 = 31.1% | 70.1% |
| **Total** | — | **24/98 = 24.5%** | — |

### Per-speaker RMS distributions (why it's hard)

| Video | Wearer mean | Other mean | Gap | Overlap |
|---|---|---|---|---|
| timur_myles_2 | 0.043 | 0.015 | 0.028 | moderate |
| timur_peter | 0.099 | 0.047 | 0.052 | high |
| timur_will | 0.106 | 0.072 | 0.034 | very high |

The wearer and other speaker have heavily overlapping RMS distributions at the frame level, especially in `timur_will` (noisy recording). No single-threshold RMS approach can get below ~25% error.

A logistic regression trained on all available features (RMS, spectral ratio, low/high energy, VAD prob) — even overfit to training data — only achieves 71% frame accuracy and 29.6% segment error. These features are insufficient.

### What's needed to get below 10%

RMS-based classification has hit its ceiling. Need a fundamentally different signal:
- Speaker embeddings (resemblyzer, pyannote, speechbrain ECAPA-TDNN)
- Better mic hardware / noise suppression
- Visual channel (face detection already in pipeline, could help disambiguate)
